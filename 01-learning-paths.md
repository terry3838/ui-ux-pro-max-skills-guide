# 🗺️ 단계별 학습 경로 (Learning Paths)

"디자인, 대체 뭐부터 시작해야 하죠?" 🤯
막막하신 분들을 위해 준비했습니다. 아래의 3주 완성 로드맵을 따라오시면, 텅 빈 화면에서 시작해 완성도 높은 전문가 수준의 디자인을 얻으실 수 있습니다.

---

## 🚩 1주차: 뼈대와 분위기 잡기 (기초 공사)

가장 먼저 해야 할 일은 내 제품의 **'성격'과 '첫인상'**을 결정하는 일입니다.

### 1단계: 어떤 스타일로 만들까요?
- **참고 문서**: `categories/ui-styles.md`
- **할 일**: 내 제품이 신뢰감을 줘야 하는 금융 앱인지, 톡톡 튀는 뷰티 앱인지 결정하세요. 문서에서 '미니멀리즘', '글래스모피즘' 등 가장 어울리는 스타일 1가지를 선택합니다.
- **AI 프롬프트 예시**: *"우리 서비스는 B2B 업무용 툴이야. ui-styles에 있는 '미니멀리즘(Minimalism)' 스타일을 적용해서 디자인 컨셉을 잡아줘."*

### 2단계: 브랜드 컬러와 폰트 정하기
- **참고 문서**: `categories/colors-fonts.md`
- **할 일**: 문서에 있는 161개의 컬러 조합과 57개의 폰트 조합 중 내 서비스와 어울리는 짝을 찾습니다. 
- **AI 프롬프트 예시**: *"신뢰감을 주는 파란색 톤의 SaaS 팔레트와, 가독성이 좋은 '현대적이고 전문적인(Modern Professional)' 폰트 조합을 세팅해 줘."*

---

## 🚩 2주차: 화면 조립하기 (인테리어)

스타일과 색상을 정했으니, 이제 실제 화면에 들어갈 요소들을 조립할 차례입니다.

### 3단계: 전체 레이아웃 구성하기
- **참고 문서**: `categories/products-landing.md`
- **할 일**: 내가 만들려는 것이 '제품 소개 랜딩 페이지'인지 '관리자 대시보드'인지 확인하고, 필수적으로 들어가야 할 섹션(헤더, 영웅 섹션, 기능 소개 등)을 배치합니다.
- **AI 프롬프트 예시**: *"products-landing 문서를 참고해서, 우리 서비스의 '이커머스 랜딩 페이지' 뼈대를 구성해 줘. 상단에 히어로(Hero) 섹션이 꼭 들어가야 해."*

### 4단계: 컴포넌트(버튼, 카드 등) 디테일 살리기
- **참고 문서**: `categories/components.md`
- **할 일**: 화면에 들어갈 버튼, 표(Table), 차트 등의 디테일을 1주차에서 정한 스타일에 맞게 디자인합니다.

---

## 🚩 3주차: 마무리와 사용자 배려 (품질 검수)

예쁘기만 한 디자인은 좋은 디자인이 아닙니다. 사용자가 쓰기 편해야 합니다.

### 5단계: UX 법칙 점검하기
- **참고 문서**: `categories/ux-guidelines.md`
- **할 일**: 완성된 화면이 문서에 나온 '절대 하면 안 되는 행동(Don't)'을 범하고 있지는 않은지 점검합니다.
- **AI 프롬프트 예시**: *"지금까지 만든 UI 코드를 ux-guidelines 문서의 기준에 맞춰서 리뷰해 줘. 접근성이나 내비게이션에 문제가 없는지(Do/Don't) 검사해 줘."*

🎉 **축하합니다!** 이 3단계를 모두 거치셨다면, 훌륭한 UI/UX를 갖춘 제품이 탄생했을 것입니다!
